# Narrative Design v2 — Gate2Reality

> Версия: 2.0 · Дата: 2026-06-22  
> Статус: **Утверждён** — заменяет встроенный сценарий v1 (стул→книга→чашка)

---

## Концептуальный сдвиг

**v1:** три фиксированных объекта → линейный квест → портал  
**v2:** любой объект в любой комнате → YOLO видит мир → существо реагирует → процедурная история разворачивается через исследование

Игра не навязывает сцену. Сцена — это реальная комната игрока.

---

## Персонаж v1: Барабашка

### Кто он

Мелкий (≈40 см) древний полтергейст. Не злой по природе — злой от скуки и одиночества.  
Живёт в пространстве, а не привязан к конкретному предмету. Мстит за невнимание. Пугает не чтобы навредить — чтобы его заметили.

**Центральное противоречие:** хочет чтобы его нашли, но прячется когда смотришь прямо.  
Это и есть ядро всей механики.

> **Реализация v1:** один архетип на сессию. Будущие версии расширят пул до 4–5 архетипов (Скиталец, Мимик, Страж, Голодный) с рандомным выбором при старте.

## AR-оболочка: принцип «некомфортной достоверности»

Чем ближе к реальности — тем страшнее. Не светящийся призрак, а нечто почти неотличимое от мира, в котором что-то не так.

### Уровни присутствия (нарастают с количеством триггеров)

| Уровень | Триггеры | Что видит игрок |
|---------|----------|-----------------|
| L0 — спящий | 0 | Ничего визуального. Только звук: тихий стук, скрип |
| L1 — тень | 1–2 | Тёмное пятно ≈40 см, движется по полу позади объекта. Видно только краем кадра |
| L2 — силуэт | 3–4 | При быстром повороте камеры — размытый контур у стены на долю секунды. При прямом взгляде замирает |
| L3 — касание | 5–7 | На поверхностях рядом с распознанными объектами — деформация текстуры, как будто что-то давит изнутри |
| L4 — лицо | 8+ | В отражающих поверхностях (экран, окно, зеркало — если YOLO распознаёт) на долю секунды появляется другое лицо рядом с игроком |

### Правила визуального поведения

- **Никогда не появляется в центре кадра** при прямом взгляде — только периферия, только краем
- **Движется когда не смотрят** (AvertedGaze), **замирает когда смотрят** — это физический закон барабашки
- **Инерция:** следует за движением камеры с задержкой 0.3–0.5 с — как реальное живое существо
- **Цветовая коррекция:** -3% насыщенности вокруг его позиции, не больше — едва заметно

## Расширенная система триггеров

**Принцип:** любой объект который YOLO распознала — потенциальный триггер.  
Нет фиксированного порядка. Нет обязательных объектов. Игрок сам исследует свою комнату.

### Группировка по «памяти»

Барабашка помнит объекты через призму того что с ними делали люди:

| Группа | YOLO-классы (примеры) | Тема для Gemma |
|--------|----------------------|----------------|
| **Сон** | bed, couch, pillow, blanket | Что происходит ночью. Кто ходит. Что снилось |
| **Еда** | table, cup, bowl, bottle, fork | Кто ел. Что оставляли. Что пропало со стола |
| **Движение** | door, window, stairs, bicycle | Кто входил. Кто не вышел. Что приносили |
| **Ребёнок** | teddy bear, ball, backpack | Дети в этом месте. Их игры. Их страхи |
| **Свет** | lamp, tv, laptop, phone | Что видели экраны. Что показывали в темноте |
| **Острое** | knife, scissors | Случаи. Барабашка говорит косвенно, не напрямую |

### Нарративная стадия

```
Триггер #1        → ЗАВЯЗКА   (первое воспоминание)
Триггеры #2–4     → РАЗВИТИЕ  (ссылки на предыдущие объекты)
Триггеры #5–7     → НАГНЕТАНИЕ (барабашка становится конкретнее, личнее)
Триггеры #8+      → КРИЗИС    (история движется к развязке)
```

Gemma получает в контекст **последние 2–3 триггера** — для связности. Каждый новый фрагмент либо упоминает предыдущий объект, либо сознательно противоречит ему.

## Ритуальные механики

Три механики отобраны как наиболее органичные для AR без геймпада:

### Ритуал 1 — Обход (Encirclement)

**Действие:** игрок медленно обходит объект вокруг, держа камеру на нём. YOLO должна удерживать объект ≥270° полного оборота.

**Эффект:** барабашка «зеркалит» — его тень делает то же самое, но в обратную сторону и с опозданием. Gemma генерирует самый длинный фрагмент за сессию — «главный секрет» объекта.

**Почему работает:** использует реальное физическое движение. Никакого UI.

---

### Ритуал 2 — Молчание (Sustained Stillness)

**Действие:** игрок держит камеру абсолютно неподвижно на объекте 8 секунд (gyroscope < порог).

**Прогрессия:**
- 3 с → объект начинает «дышать» — микродеформация геометрии
- 6 с → звук (скрип, шорох рядом)
- 8 с → барабашка появляется рядом с объектом на ≈0.5 с, уходит

Это **единственный момент** когда его можно почти увидеть прямо.

**Почему работает:** рука устаёт держать неподвижно — реальный физический дискомфорт усиливает напряжение.

---

### Ритуал 3 — Совмещение (Alignment)

**Действие:** игрок помещает два объекта из разных групп в один кадр одновременно.

**Эффект:** пара активирует уникальную нарративную ветку. Барабашка реагирует агрессивнее или испуганнее в зависимости от комбинации.

**Примеры пар:**

| Пара | История |
|------|---------|
| `lamp` + `phone` | Что видели ночью когда думали что никто не смотрит |
| `teddy bear` + `scissors` | История которую барабашка не хочет рассказывать до конца |
| `door` + `bottle` | Кто приходил и в каком состоянии |
| `bed` + `knife` | Обрывается на полуслове |

**Почему работает:** игрок сам «составляет» историю через выбор объектов — реальная агентность.

## Моральный выбор — контекстуальный финал

Когда накоплено ≥5 триггеров и пройден хотя бы один ритуал — барабашка «показывает» что ему нужно. Не через UI — через AR-действие.

Gemma анализирует накопленный контекст сессии и выбирает один из трёх финальных архетипов:

| Контекст истории | AR-действие игрока | Финал |
|------------------|--------------------|-------|
| Барабашка потерял что-то конкретное | Навести камеру на тот объект и держать 10 с — «вернуть» | Уходит. Тишина. Комната обычная. |
| Привязан к месту насилием | Выйти из комнаты — дверь в кадр, физически уйти | Отпускает пространство. Звук удаляется. |
| Просто одинок | Положить телефон экраном вниз — «принять соседство» | Остаётся. Иногда двигает предметы. |

## Архитектура промпта Gemma

### System prompt (фиксированный для архетипа)

```
Ты — барабашка. Тебе несколько сотен лет.
Ты маленький, мохнатый, невидимый когда захочешь.
Ты живёшь в этом пространстве и помнишь всё что здесь происходило с людьми.
Ты говоришь коротко — 1-2 предложения максимум.
Иногда обрываешь фразу на полуслове. Иногда рифмуешь — не специально, просто вышло.
Ты не злой — но тебя злит когда тебя игнорируют.
Ты помнишь конкретных людей и конкретные события связанные с предметами.
Каждый раз придумывай их разными. Не повторяй детали из предыдущих фраз.
Говори от первого лица.
Запрещённые слова: призрак, дух, смерть, убийство, монстр, ужас.
```

### User prompt (каждый триггер)

```
Объект: {yolo_label_ru}
Предыдущие объекты этой сессии: {context_chain}
Стадия истории: {завязка | развитие | нагнетание | кризис}
Освещение: {тёмно | нормально | светло}
Напиши 1-2 предложения от первого лица про этот объект.
```

### Контекстная цепочка (пример)

```
стол → кружка → лампа → [текущий: нож]

Объект: нож
Предыдущие: стол, кружка, лампа
Стадия: нагнетание
Освещение: тёмно
```

Gemma ссылается на предыдущие объекты — это создаёт ощущение живой памяти, а не набора случайных фраз.

## Исправленные логические провалы

| Провал v1 | Решение v2 |
|-----------|------------|
| Зачем игрок смотрит на объекты? | Игра начинается со звука (стук, скрип) до первого триггера. Игрок поднимает телефон сам. |
| Почему стул→книга→чашка? | Нет фиксированного порядка. YOLO находит что найдёт в любой комнате. |
| Guard Node без нарратива | Барабашка злится от игнорирования — это его характер. 45 с тишины = он начинает действовать. |
| Карта с зонами появляется произвольно | Барабашка показывает свою территорию — места где его присутствие сильнее всего. |
| Зачем переходить в Главу II? | Барабашка ведёт игрока к себе. Игрок буквально входит на его сторону реальности. |
| Игрок — пассивный свидетель | Три ритуала + финальный выбор дают реальную агентность. |
