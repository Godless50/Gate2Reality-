# Глава II «По ту сторону» — дизайн и каркас

Та же физическая комната игрока — но правила перевёрнуты. Глава II построена
**переиспользованием ядра**: ни одной правки FSM графа, только новая семантика
поверх существующих событий + один новый тип условия.

Точка входа: `CrossingTransitionEffect.OnCrossedOver` (игрок под пиком вспышки
пересёк портал в финале Главы I).

---

## Каркас (namespace `Gate2Reality.ChapterTwo`)

| Файл | Роль |
|------|------|
| `ChapterTwoDirector.cs` | Загрузчик главы по `OnCrossedOver`: включает корень Главы II, переносит якоря комнаты в узлы, стартует Ch2-граф |
| `InvertedGuardDirector.cs` | Перевёрнутый Guard: те же события ядра — адверсариальные реакции |
| `InvertedObjectEffect.cs` | «Знакомые объекты ведут себя иначе»: предмет оживает, крадётся, пока не смотрят |

Плюс расширение ядра: `ConditionType.AvertedGaze` в `NarrativeCondition`.

---

## Три перевёрнутых правила

### 1. Guard помогает ПРОТИВ игрока
В Главе I `NarrativeManager` при простое 45с→60с→75с СПАСАЛ игрока: маяк →
десатурация → партиклы-проводник. FSM ядра в Главе II **не меняется** —
`InvertedGuardDirector` переопределяет только смысл реакций:

| Событие ядра | Глава I (спасение) | Глава II (охота) |
|--------------|--------------------|------------------|
| `OnAudioBeaconRequested` | 3D-маяк у цели | **ложный зов** из точки за спиной игрока — уводит прочь |
| `OnDesaturateRequested` | обесцветить фон | **мир сжимается**: виньетка наезжает, изнанка давит |
| `OnSaturationRestoreRequested` | вернуть краски | отпускает хватку при прогрессе |
| `OnGuideParticlesRequested` | поток камера→цель | **тянущиеся** партиклы цель→игрок (комната тянет к себе) |

Простой = опасность. Чистая инверсия без правок ядра.

### 2. Знакомые объекты ведут себя иначе
`InvertedObjectEffect` (наследник `TriggerableEffectBase`): узел Главы II
срабатывает — и привычный предмет «оживает». В отличие от one-shot эффектов
Главы I **не** зовёт `MarkFinished`, остаётся активным:
- `watchPlayer` — медленно разворачивается лицом к игроку (следит);
- `creepToward` — подкрадывается, останавливаясь на `minDistance`;
- `onlyWhenUnobserved` — движется ТОЛЬКО пока на него не смотрят (замирает
  под взглядом);
- `distortRamp` — локально поднимает `_BreachProgress` через
  `MaterialPropertyBlock` (без инстансинга материала).

### 3. Перевёрнутый взгляд — `ConditionType.AvertedGaze`
Новый тип условия (единственная правка ядра, аддитивная). Тематическая инверсия
`Gaze`: узел срабатывает, когда якорь рядом (`maxGazeDistance`), но игрок смотрит
В СТОРОНУ (за пределами конуса `maxGazeAngleDeg`). Та же дешёвая математика
(один `Dot`, без `acos`), обратный предикат. «Объект живёт, пока за ним не
наблюдают» — на уровне графа, а не хардкода.

## Поток включения

```
CrossingTransitionEffect.OnCrossedOver
        │
        ▼
ChapterTwoDirector.HandleCrossedOver   (строго одноразово)
        │
        ├─ chapterTwoRoot.SetActive(true)      // Ch2 NarrativeManager.Awake (без автостарта)
        ├─ invertedSideVolume.weight = 1        // база изнанки
        ├─ invertedAmbience.Play()
        ├─ detector.SetPersonOnlyMode(true)     // обет приватности продолжается
        ├─ ApplyCarriedAnchors()                // якоря комнаты Главы I → узлы Главы II
        └─ chapterTwoNarrative.StartScene()     // старт с уже привязанными целями
        │
        ▼
   OnChapterTwoBegan
```

**Почему отдельный NarrativeManager:** Глава II — это свой граф-ассет
(`NarrativeGraphAsset`, авторится в новом редакторе графа из Этапа E). Корень
Главы II в сцене выключен; его менеджер с `autoStartOnStart=false`, чтобы
директор успел привязать перенесённые якоря ДО входа в пространственные узлы.

---

## Privacy в изнанке
Обет «гасим хоррор при людях в кадре» действует всю игру. Детектор остаётся в
person-only (1 Гц, ~0.2 Вт); Глава II работает на пространственных условиях
(`Proximity` / `AvertedGaze`) по перенесённым якорям, а не на тяжёлом YOLO.
Глобальный `_HorrorScale` (`HorrorSafetyGovernor`) гасит и дисторсию
`InvertedObjectEffect` — предметы «успокаиваются» при появлении человека.

## Что ещё нужно (продакшн Главы II)
- Ch2 граф-ассет (узлы AvertedGaze/Proximity на якорях комнаты) — в редакторе;
- ARCore persistent anchors: перенести точные позы объектов Главы I между
  сессиями (сейчас якоря передаются трансформами в пределах одной сессии);
- Ассеты изнанки: пост-профиль (холодный LUT, наезжающая виньетка для сжатия),
  «неправильные» аудио-кью объектов, партиклы тянущихся рук;
- Сейв `crossedOver` уже пишется `ProgressTracker` — добавить ветку resume
  прямо в Главу II.
